Задание: Разработка приложения Todo List

В рамках задания реализован следующий функционал:
- Отображение списка задач;
- Добавление новых задач;
- Отметка задач как выполненных;
- Удаление задач;
- Редактирование существующих задач;
- Фильтрация задач по статусу:
	- Все
	- Активные
	- Выполненные
- Добавлена типизация (TypeScript);
- Реализована возможность изменения порядка задач с помощью drag-and-drop (для этого использована библиотека react-beautiful-dnd);
- Настроено сохранение состояния в локальном хранилище браузера для сохранения данных между сессиями (использована библиотека redux-persist).

Структура проекта
Проект организован по принципу разделения ответственности.
📁 shared
Содержит общие и переиспользуемые ресурсы:

- shared/assets — иконки;
- shared/types — типы данных, интерфейсы, перечисления (Enum);
- shared/ui — универсальные UI-компоненты (Button, Checkbox, Field).

📁 features
Содержит функциональные модули:
- fieldAction — поле ввода задачи и кнопка добавления;
- filter — фильтрация задач по статусу (все / активные / выполненные);
- task — компонент отдельной задачи.

📁 widgets
ToDo — основной виджет, объединяющий всю логику и интерфейс приложения.

📁 pages
В данной папке отсутствует бизнес-логика.
Здесь размещается только отображение заголовка страницы и подключение виджета ToDo.

📁 entities
Содержит логику работы с состоянием:
- slice;
- reducer;
- actions.
